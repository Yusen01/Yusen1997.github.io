左式堆(leftist heap)与斜堆(skew heap)

发表于 2017-05-03 | 分类于 数据结构

左式堆(leftist heap)

(一)定义与性质

我们先来看看左式堆和二叉堆的异同：

相同点:

  • 左式堆和二叉堆都是一种堆结构。
  • 左式堆和二叉堆都具有堆结构的两个基本属性——结构性和有序性。

不同点:

  • 二叉堆是一个完全二叉树，因此非常的平衡。而左式堆则要求整棵树尽可能不平衡，因此不是一个完全二叉树。
  • 由于左式堆不是一个完全二叉树，因此必须使用指针来构建，而不能简单的使用数组。

大致了解了左式堆和二叉堆的异同后，我们接着看几个基本概念。

npl(null path length): 一个结点的npl等于从该结点到一个没有两个孩子的结点的最短路径。

在左式堆中，一个结点的左孩子的npl必须大于或等于右孩子的npl。

因此，空结点的nol为-1，单结点或只有一个孩子的结点的npl为0，有两个孩子的结点的npl等于它的右孩子的npl加1。

right path: 从根结点出发，一直向右结点搜索，直到遇到空结点，所得到的路径就是right path。

为什么要有right path这个概念？我们在之前提到有两个孩子的结点的npl等于它的右孩子的npl加一。因此我们要计算一个结点的npl，就要计算右孩子的npl，而要计算右孩子的npl，就要计算右孩子的右孩子的npl…一直迭代下去直到遇到右孩子为空结点的结点。而整个计算的路径其实就是所谓的right path，一个结点的npl其实就是它的right path的路径长度。

接下来我们给出一个定理:

一个right path上共有r个结点的左式堆，它至少含有2^r - 1个结点。

使用数学归纳法可以对其进行简单证明:

  • basic step: 当r等于1时，显然至少含有一个结点。
  • iductive step: 假设对于1,2,3…..r，上述定理均成立。当right path上有r + 1个结点时，根结点的右子树的right path共有r个结点。由于一个结点的npl等于right path的路径长度，也就是结点数减1，因此右结点的npl为r - 1。又因为左结点的npl要大于等于右结点的npl，因此左子树的right path也至少含有r个结点。所以根据假设，左子树的结点数至少为2^r - 1，右子树的结点数也至少为2^r - 1，两者相加再加上根结点后，至少包含2^(r + 1) - 1个结点，得证。

由n >= 2^r - 1，我们可以得到r <= log(n + 1)。因此right path上的结点数为O(log n)，我们将在其上进行一系列操作。

(二)相关操作

左式堆中最重要的一个操作就是合并(merge)操作。

假设存在两个左式堆H1，H2，均为最小堆。

1、merge

  • 将两个树中的根结点较小者作为新的根结点。比如H1->val < H2->val，则H1的根结点即为新的根结点。
  • 保留新的根结点的左子树。
  • 递归合并新的根结点的右子树和另一个左式堆，将其作为新的右子树。上例中即为合并H1的右子树和H2，得到的新的树作为H1的右子树。
  • 从right path的最后一个元素开始根据npl值逐步调整整棵树，使最终结果满足左式堆性质。

2、insert

插入操作可以看成一个左式堆和一个单结点的左式堆合并。

  • 构造一个单结点的左式堆。
  • 将其与另一个左式堆合并。

3、deletemin

删除最小值操作也非常的简单。

  • 删除根结点。
  • 合并左右子树。
(三)时间复杂度

merge操作是对right path上的点进行操作，因此时间复杂度为O(log n)。

insert操作基本也是一个合并操作，时间复杂度为O(log n)。

deletemin操作也是一个合并操作，时间复杂度为O(log n)。

斜堆(skew heap)

斜堆相比于左式堆来说更为简单，两者之间的区别在于:

  • 斜堆没有npl概念。
  • 合并操作时，对于所有right path上的点，我们都需要交换它的左右子树。

斜堆的时间复杂度依靠摊还分析来支撑，每一个操作的摊还时间均为O(log n)。

B树与B+树

发表于 2017-03-31 | 分类于 数据结构

转载请注明出处。www.huangyusheng.com

B树和B+树是我一直没搞懂的数据结构，今天花了一些时间，大概理清楚了基本的知识，写在博客里以记录。

1、B树

(一)定义

其实一开始自己搞不懂B树，是因为B树的定义比较模糊。关于B树的定义本来就不统一，加上网上许多博客在写关于B树的技术资料时有许多错误（可能也不算错误，但确实有些混淆），于是就越来越搞不懂了。但真正理解定义之后，B树也就不算太难了。

第一种是用阶来定义B树

根据维基百科，一个m阶的B树定义为：

  • 根结点最少有2个孩子结点，最多有m个孩子结点。
  • 每一个非根非叶结点最少有⌈m/2⌉(m/2取上限)个孩子结点，最多有m个孩子结点
  • 所有有k个孩子结点的非叶子结点有k - 1个关键字
  • 所有的叶子结点都在同一层

内部结点是指根结点和叶子结点之外的结点，它有至少⌈m/2⌉(m/2取上限)个孩子，至多有m个孩子。

根结点至少有2个孩子，至多有m个孩子。

叶子结点没有孩子结点和指向孩子结点的指针，但关键词的个数受到和内部结点一样的限制，即关键词个数在⌈m/2⌉ - 1 和m - 1之间。

第二种是用度来定义B树

阶的定义是用孩子结点的个数限制关键词个数，而度的定义则是用关键词的个数来限制孩子结点的个数。

根据算法导论中，一个最小度数为t的B树定义为：

  • 每个结点包含的关键词的个数有一个上界和一个下界
  • 除了根结点之外的所有结点都至少含有t - 1个关键词，至多含有2t - 1个关键词
  • 根结点至少含有1个关键词，至多含有2t - 1个关键词
  • 其他定义和阶定义相同
(二)树的高度

关于B树的高度，算法导论有相应的结论以及证明。

(三)B树的插入

B树的插入操作还是比较简单的。大致步骤如下：

  • 首先根据插入的关键字，在B树中搜索，找到对应的叶子结点
  • 将关键字放入叶子结点
  • 放入后，如果该结点的关键字个数超过了限制，比如大于2t - 1，则将该结点分裂，提出中间关键字将其放入父结点。
  • 如果父结点的关键字个数也超过了限制，那么继续分裂，按此步骤一直执行，直到所有结点都满足B树的定义。最坏的情况下，可以一直分裂到根结点，最终树高加1。

(四)B树的删除

B树的删除操作大致步骤如下：

  • 找到要删除的关键字
  • 如果该关键字有两个孩子，则先删除关键字，然后从结点数较多的孩子中取一个关键字(左子树取最大的，右子树取最小的)代替被删除的关键字的位置。如果该关键字没有孩子，则直接删除关键字。
  • 删除关键字之后，如果某一结点的关键字个数小于最小关键字个数，比如小于(m/2取下限)，则需要对其调整。如果该结点的兄弟结点的关键字个数大于最小关键字个数(不能等于)，则向父结点借一关键字放入该结点，并从兄弟结点中取一关键字放入父结点。如果兄弟结点的关键字个数也都刚好等于最小关键字个数，则需要将该结点和某一兄弟结点合并。
  • 如果合并之后造成父结点的关键字个数小于最小关键字个数，则对父结点执行步骤三，以此类推。最坏的情况下，一直合并到根结点，树的高度减一。

(五)B树的意义

谈到B树，大家都会说B树适合文件系统和数据库，在磁盘操作上有很大优势。但一开始的时候我就搞不明白为什么B树会有这些优点，感觉它跟普通的树也没啥差别啊，不就是一个结点可以有很多个孩子，树比较矮吗，咋就有利于磁盘IO了呢？

其实我犯了一个比较蠢的错误，我一直以为B树的储存就像普通的树一样，都是放在内存中的，只不过结点中的关键字对应有一个指向所谓“卫星数据”的指针。因此我一直不理解为什么B树的高度低会有利于磁盘读写(因为理解成在内存中储存结构的话，搜索关键字的过程是不需要磁盘读写的)。

后来我才意识到(可能是自己比较笨)，原来B树之所以厉害，是因为它的所有结点并不是储存在内存中，而是放在磁盘上，这样一来，每个结点就能储存大量的关键字信息而不受内存大小的限制，因此我们每访问一个结点都需要对磁盘进行一次读写操作，而B树的高度较低，因此磁盘读写的次数就非常的少。正是因为B树能够储存大量的关键字信息，并且访问时需要较少的磁盘IO，因此才适合在文件系统和数据库中使用。

2、B+树

B+树是B树的一个变种，它和B树不同点在于：

  • B+树中所有的关键字信息储存在叶子结点，非叶子结点只做为索引。
  • B+树的叶子结点分裂时，中间关键字不会从原来的结点删除，而是保留在分裂后产生的右结点中。但其他非叶子结点分裂则和B树相同。
  • B+树的删除只能从叶子结点删除，删除之后再进行调整。(B+树的删除操作的具体规则我暂时也不是很清楚，留着以后解决)

B+树和B树另一个不同点就是B+树中的叶子结点是链接在一起的，像是一个链表，如下图所示。

B+树的优点

  • 非叶子结点只包含关键字，不包含指向“卫星数据”的指针，因此如果一个结点的磁盘空间是固定的，那么一个结点可以储存更多的关键字。
  • 由于B+树中所有信息都储存在叶子结点，而这些叶子结点又是相互连接的，因此可以很方便的遍历所有关键字，而在B树中只能采用中序遍历。

B+树的这些优点使得B+树比B树更适合数据库系统或文件系统。

红黑树(red-black tree)

发表于 2017-03-14 | 分类于 数据结构

转载请注明出处。www.huangyusheng.com

1、定义

首先红黑树是一个二叉搜索树，因此满足二叉搜索树的所有性质。其次，红黑树也有自己特殊的五个性质。

  • 所有结点要么是红色，要么是黑色
  • 根结点是黑色的
  • 所有的叶子结点是黑色的（注意，这里的叶子结点是指空结点，即NULL结点）
  • 红结点的两个孩子都是黑色的
  • 对于任意一个结点，它到其各个叶子结点的路径上的黑结点数量都相同

这些看起来非常复杂的性质保证了一颗含有n个结点的红黑树，它的高度至多为2log(n+1)。具体的证明请参考算法导论第三版第13章。

对于一个高度为O(log(n))的二叉搜索树，显然插入、删除、查找等动态集合操作的时间复杂度均为O(log(n))。

2、插入

红黑树的插入操作还算比较简单，只需要改变一些颜色和简单的旋转就可以完成。由于左右对称，我们这里就只讨论左边的情况。

红黑树的插入操作和二叉搜索树类似，它也需要不断搜索找到一个空位，然后把自己插进去。因为结点都是带颜色的，所以新插入的结点也要有一个颜色。那么是给它涂上红色好呢还是涂上黑色好呢？从第五个性质我们就可以判断出一定是涂上红色好！因为涂上红色有时候不需要做任何的调整，而涂上黑色则第五个性质一定会被破坏，我们肯定要对整个树进行调整，所以涂红的能占点便宜，我们当然选择涂红的啦。

因此，对红黑树插入一个新结点，我们需要按照二叉搜索树的插入方法找到一个空位，把结点插入到空位上，并给它涂上红色。

接下来我们就需要对整个树进行调整，以使整个树满足红黑树的性质。我们先来看两种简单的情况：

  • 原来的树是一个空树，插入的结点成为根结点。这种情况下我们只要把结点变成黑色就可以了。
  • 插入结点的父节点是一个黑结点。这种情况下没有任何性质被破坏，因此啥都不用做。

如果插入的结点不满足上面两种情况，那么它的父节点一定是一个红结点，并且它有祖父结点（因为根节点是黑色的）。这种情况下，我们为了调整，需要观察它的叔父结点（也就是祖父结点的右孩子），当然叔父结点可能是个空结点，我们把它当成普通的黑结点处理即可。

1、叔父结点是红结点

这种情况下，我们只需要将父节点和叔父结点变为黑色，祖父结点变为红色，然后将祖父结点作为新的待调整的结点（上移两层）。

如图所示，z结点表示新插入的结点，y结点表示它的叔父结点，new z结点表示新的待调整结点。

2、叔父结点是黑结点，并且新插入节点是它的父节点的右结点

这种情况下，需要对父节点做一个左旋，转换成第三种情况。

3、叔父结点是黑结点，并且新插入结点是它的父节点的左节点

这种情况下，我们要将父节点变为黑色，祖父结点变为红色，然后对祖父结点做右旋。

以上就是插入操作，接下来我们来探讨一下删除操作。

3、删除

对于一个二叉搜索树的删除操作有很多中不同的算法。比如我之前习惯用递归的方法来做删除，先找到要删除的值，如果没有右子树就直接删掉，如果有再从右子树中找最小值，把最小值拷贝到当前节点，然后在右子树中递归删除这个值。在算法导论中，对二叉搜索树的删除操作是这样的，先找到要删除的值，如果没有子树，则直接删除，如果有一个子树，则用子树的根结点替换当前的结点，如果有两个子树，则在右子树中找到最小值，用最小值结点的右子树替换最小值结点，再用最小值结点替换要删除的结点。

红黑树的删除操作也和二叉搜索树类似，但在删除后对树做调整以满足红黑树的性质。在实际编程中我们需要用一个循环来一步一步的调整整个树，如果使用递归的话编程可能会更加困难。因此，我们使用算法导论中的删除方法来做红黑树的删除操作。

如果删除的结点有两个子树，我们找到右子树中的最小值结点，并用这个结点的右节点替换这个结点，再用这个结点替换要删除的结点，在替换要删除的结点时只要把要删除结点的颜色给这个最小值结点，任何一条性质都不会被破坏，只有原来最小值结点的右节点可能会破坏红黑树的性质。因此，不管要删除的结点有一个子树还是有两个子树，都可以归结为删除一个只有一个子树的结点。

接下来，我们考虑两种简单情况：

  • 被删除的结点是红结点。这种情况不需要做任何操作。
  • 被删除的结点是黑结点，但替换它的结点是红结点。这种情况下，我们只需要将替换结点变为黑色即可。

如果不满足上面的两种情况，说明被删除的结点是黑色的，同时替换结点也是黑色的。为了保持性质五不被破坏，我们将替换结点看作有两重黑色。接着我们需要考虑替换结点的兄弟结点。由于左右对称，我们这里只讨论替换结点是父节点的左孩子的情况。

1、兄弟结点是红结点

这种情况下，我们要将父节点变为红色，兄弟结点变为黑色，并对父节点做左旋。将情况转为下面的几种情况。

2、兄弟结点是黑结点，并且它的两个孩子都是黑结点

这种情况很简单，将两重黑色提取一重出来和兄弟结点的黑色一起赋给父节点，原来拥有两重黑色的结点变为只有一重黑色，兄弟结点变为红色，父节点变为两重颜色（可能是红黑也可能是黑黑），将父节点作为新的待调整结点。

为什么一定要兄弟结点的两个孩子都是黑结点呢？因为兄弟结点在被提取黑色之后会变为红色，如果它的子节点有红色的话，红黑树性质4会被破坏，调整会变得更加复杂。

3、兄弟结点是黑结点，并且它的左孩子是红结点，右孩子是黑结点

这种情况下，我们需要将兄弟结点变为红色，它的左孩子变为黑色，然后对兄弟结点右旋，将情况转为下一种情况。

4、兄弟结点是黑结点，并且它的右孩子是红结点

这种情况下，我们要将父节点变为黑色，兄弟结点变味红色，对父节点做左旋使得兄弟结点变为子树的根结点，然后将兄弟结点的右孩子涂黑。过程有些复杂，看图会更清楚一些。

以上就是红黑树的删除操作。

4、总结

说了这么多，其实都是纸上谈兵，我们只能在某种抽象的层面上了解红黑树这种数据结构和相应的算法，但真要实现一个红黑树无疑是一个很复杂的事，许多高级数据结构都是这样，看得懂但写不出。所以我没有放任何代码在文章里，一是觉得放了也没人会看。二是我自己也还写不出。。。。。不管怎么样，能多了解些数据结构也是一件有益无害的事，对吧！

PAT甲级1127 ZigZagging on a Tree(C++)

发表于 2017-03-13 | 分类于 PAT甲级

PAT甲级1127 ZigZagging on a Tree，原题地址：https://www.patest.cn/contests/pat-a-practise/1127
解题分析：
用后序遍历序列和中序遍历序列构建一个二叉树，然后层序遍历，这个zigzag输出也很简单，判断一下层数是奇数还是偶数就可以了。。。

#include&lt;iostream&gt;
#include&lt;vector&gt;
#include&lt;queue&gt;
using namespace std;

struct node{
    int key;
    struct node *left;
    struct node *right;
};

int preOrder[50];
int postOrder[50];
int n;
int Index;

// 构建二叉树
struct node *Construct(int start , int end){
    struct node *ptr;
    int i;

    if(start &gt; end)
        return NULL;
    ptr = new struct node;
    ptr-&gt;key = postOrder[Index];
    for(i = start;i &lt;= end;i++){
        if(preOrder[i] == postOrder[Index]) 
            break;
    }
    Index--;
    ptr-&gt;right = Construct(i + 1 , end);
    ptr-&gt;left = Construct(start , i - 1);
    return ptr;
}

// 层序遍历
void LevelOrder(struct node *root){
    int cnt = 1;
    int flag = 1;
    queue&lt;struct node *&gt; q;
    vector&lt;int&gt; v;

    q.push(root);
    while(!q.empty()){
        int t = q.size();
        v.clear();
        while(t &gt; 0){
            struct node *ptr = q.front();
            q.pop();
            if(ptr-&gt;left != NULL)
                q.push(ptr-&gt;left);
            if(ptr-&gt;right != NULL)
                q.push(ptr-&gt;right);
            t--;
            v.push_back(ptr-&gt;key);
        }
        if(cnt % 2 != 0){
            for(int i = v.size() - 1;i &gt;= 0;i--){
                if(flag){
                    flag = 0;
                }
                else{
                    cout &lt;&lt; " ";
                }
                cout &lt;&lt; v[i];
            }
        }
        else{
            for(int i = 0;i &lt; v.size();i++){
                if(flag){
                    flag = 0;
                }
                else{
                    cout &lt;&lt; " ";
                }
                cout &lt;&lt; v[i];
            }
        }
        cnt++;
    }
}
int main(){
    int i;
    struct node *root = NULL;

    cin &gt;&gt; n;
    for(i = 0;i &lt; n;i++){
        cin &gt;&gt; preOrder[i];
    }
    for(i = 0;i &lt; n;i++){
        cin &gt;&gt; postOrder[i];
    }
    Index = n - 1;
    root = Construct(0 , n - 1);
    LevelOrder(root);
}

PAT甲级1126 Eulerian Path(C++)

发表于 2017-03-13 | 分类于 PAT甲级

PAT甲级1126 Eulerian Path，原题地址：https://www.patest.cn/contests/pat-a-practise/1126
解题分析：
判断是否是欧拉回路。首先要判断图是否是联通的，如果不联通则输出Non-Eulerian，如果联通在判断度为奇数的结点数量。

#include&lt;iostream&gt;
using namespace std;

int n , m;
int G[510][510];
int visited[510];

void DFS(int index){
    int i;

    visited[index] = 1;
    for(i = 1;i &lt;= n;i++){
        if(visited[i] == 0 &amp;&amp; G[index][i] == 1)
            DFS(i);
    }
}

int main(){
    int i , j;
    int degree;
    int cnt = 0;

    cin &gt;&gt; n &gt;&gt; m;
    for(i = 0;i &lt; m;i++){
        int a , b;
        cin &gt;&gt; a &gt;&gt; b;
        G[a][b] = 1;
        G[b][a] = 1;
    }
    // 计算输出每个结点的度
    for(i = 1;i &lt;= n;i++){
        degree = 0;
        for(j = 1;j &lt;= n;j++){
            if(G[i][j] == 1){
                degree++;
            }
        }
        // 度为奇数
        if(degree % 2 != 0){
            cnt++;
        }
        cout &lt;&lt; degree;
        if(i != n)
            cout &lt;&lt; " ";
    }
    cout &lt;&lt; endl;
    // DFS遍历判断图是否联通
    DFS(1);
    for(i = 1;i &lt;= n;i++){
        // 不联通
        if(visited[i] == 0){
            cout &lt;&lt; "Non-Eulerian" &lt;&lt; endl;
            return 0;
        }
    }
    // 判断度为奇数的结点数量
    if(cnt == 0){
        cout &lt;&lt; "Eulerian" &lt;&lt; endl;
    }
    else if(cnt == 2){
        cout &lt;&lt; "Semi-Eulerian" &lt;&lt; endl;
    }
    else{
        cout &lt;&lt; "Non-Eulerian" &lt;&lt; endl;
    }
}

PAT甲级1125 Chain the Ropes(C++)

PAT甲级1125 Chain the Ropes，原题地址：https://www.patest.cn/contests/pat-a-practise/1125
解题分析：
贪心算法。先对各部分的长度按从小到大顺序排序，每一次都连接长度最小的两段，这样就可以使长度损失降到最低，从而使得最后的长度最长。动态规划理论上也可以解，但需要内存太大了，不可行。

#include&lt;iostream&gt;
#include&lt;vector&gt;
#include&lt;algorithm&gt;
using namespace std;

int main(){
    int n;
    int i;
    vector&lt;double&gt; v;

    cin &gt;&gt; n;
    for(i = 0;i &lt; n;i++){
        int num;
        cin &gt;&gt; num;
        v.push_back(num);
    }
    sort(v.begin() , v.end());
    int ans = (v[0] + v[1]) / 2;
    for(i = 2;i &lt; n;i++){
        ans = (ans + v[i]) / 2;
    }
    cout &lt;&lt; (int)ans &lt;&lt; endl;
}

PAT甲级1123 Is It a Complete AVL Tree(C++)

发表于 2017-03-13 | 分类于 PAT甲级

PAT甲级1123 Is It a Complete AVL Tree，原题地址：https://www.patest.cn/contests/pat-a-practise/1123
解题分析：
AVL树的插入以及层序遍历，遍历时判断是否是完全二叉树。

#include&lt;iostream&gt;
#include&lt;vector&gt;
#include&lt;queue&gt;
#include&lt;cmath&gt;
using namespace std;

struct node{
    int key;
    struct node *left;
    struct node *right;
};

// 计算高度
int GetHeight(struct node *root){
    if(root == NULL)
        return -1;
    int lh , rh;
    lh = GetHeight(root-&gt;left);
    rh = GetHeight(root-&gt;right);
    return lh &gt; rh ? lh + 1 : rh + 1;
}

// 右旋
struct node *RoateWithLeft(struct node *root){
    struct node *ptr;

    ptr = root-&gt;left;
    root-&gt;left = ptr-&gt;right;
    ptr-&gt;right = root;
    return ptr;
}

// 左旋
struct node *RoateWithRight(struct node *root){
    struct node *ptr;

    ptr = root-&gt;right;
    root-&gt;right = ptr-&gt;left;
    ptr-&gt;left = root;
    return ptr;
}

// 左边两次旋转
struct node *DoubleRoateWithLeft(struct node *root){
    root-&gt;left = RoateWithRight(root-&gt;left);
    return RoateWithLeft(root);
}

// 右边两次旋转
struct node *DoubleRoateWithRight(struct node *root){
    root-&gt;right = RoateWithLeft(root-&gt;right);
    return RoateWithRight(root);
}

// 插入
struct node *Insert(struct node *root , int key){
    if(root == NULL){
        root = new struct node;
        root-&gt;key = key;
        root-&gt;left = root-&gt;right = NULL;
    }
    else{
        if(key &lt; root-&gt;key){
            root-&gt;left = Insert(root-&gt;left , key);
        }
        else{
            root-&gt;right = Insert(root-&gt;right , key);
        }
        int lh , rh;
        lh = GetHeight(root-&gt;left);
        rh = GetHeight(root-&gt;right);
        if(lh - rh &gt; 1){
            if(key &lt; root-&gt;left-&gt;key){
                root = RoateWithLeft(root);
            }
            else{
                root = DoubleRoateWithLeft(root);
            }
        }
        if(rh - lh &gt; 1){
            if(key &gt; root-&gt;right-&gt;key){
                root = RoateWithRight(root);
            }
            else{
                root = DoubleRoateWithRight(root);
            }
        }
    }
    return root;
}

// 层序遍历
void LevelOrder(struct node *root){
    int dep = 0;
    int flag = 1;
    queue&lt;struct node*&gt; q;
    vector&lt;struct node*&gt; v;

    q.push(root);
    while(!q.empty()){
        int cnt = q.size();
        int temp = cnt;
        // 当前层不满
        if(temp != pow(2 , dep)){
            // 先从上一层判断
            for(int i = 0;i &lt; v.size();i++){
                if(v[i]-&gt;left == NULL &amp;&amp; v[i]-&gt;right != NULL){
                    flag = 0;
                }
                else if(i != 0 &amp;&amp; v[i]-&gt;left != NULL &amp;&amp; v[i - 1]-&gt;right == NULL){
                    flag = 0;
                }
            }
        }
        v.clear();
        // 处理当前层结点
        while(cnt){
            struct node *ptr;
            ptr = q.front();
            q.pop();
            if(ptr-&gt;left != NULL){
                q.push(ptr-&gt;left);
                // 当前层不满，但下一层还有结点
                if(temp != pow(2 , dep)){
                    flag = 0;
                }
            }
            if(ptr-&gt;right != NULL){
                q.push(ptr-&gt;right);
                if(temp != pow(2 , dep)){
                    flag = 0;
                }
            }
            cnt--;
            // v记录上一层结点，以便以后用了判断是否完全二叉
            v.push_back(ptr);
            cout &lt;&lt; ptr-&gt;key;
            if(!q.empty())
                cout &lt;&lt; " ";
            else    
                cout &lt;&lt; endl;
        }
        // 层数加一
        dep++;
    }
    if(flag)
        cout &lt;&lt; "YES" &lt;&lt; endl;
    else
        cout &lt;&lt; "NO" &lt;&lt; endl;
}

int main(){
    int n;
    int i;
    struct node *root = NULL;

    cin &gt;&gt; n;
    for(i = 0;i &lt; n;i++){
        int t;
        cin &gt;&gt; t;
        root = Insert(root , t);
    }
    LevelOrder(root);
}

PAT甲级1122 Hamiltonian Cycle(C++)

发表于 2017-03-13 | 分类于 PAT甲级

PAT甲级1122 Hamiltonian Cycle，原题地址：https://www.patest.cn/contests/pat-a-practise/1122
解题分析：
判断给出的路径是否是包含所有结点的环。

#include&lt;iostream&gt;
#include&lt;vector&gt;
using namespace std;

int main(){
    int n , m;
    int G[210][210];
    int k , q;
    int i , j;

    cin &gt;&gt; n &gt;&gt; m;
    for(i = 0;i &lt; 210;i++){
        for(j = 0;j &lt; 210;j++){
            G[i][j] = 0;
        }
    }
    // 建图
    for(i = 0;i &lt; m;i++){
        int a , b;
        cin &gt;&gt; a &gt;&gt; b;
        G[a][b] = 1;
        G[b][a] = 1;
    }
    cin &gt;&gt; k;
    for(i = 0;i &lt; k;i++){
        int arr[210];
        int flag[210];
        // flag用来记录结点是否被访问过
        for(j = 0;j &lt; 210;j++){
            flag[j] = 0;
        }
        cin &gt;&gt; q;
        for(j = 0;j &lt; q;j++){
            cin &gt;&gt; arr[j];
        }
        // 如果输入路径的结点数不等于n＋1，那一定不是符合要求的环
        if(q != n + 1){
            cout &lt;&lt; "NO" &lt;&lt; endl;
        }
        // 如果始末位置不同，一定不是环
        else if(arr[0] != arr[q - 1]){
            cout &lt;&lt; "NO" &lt;&lt; endl;
        }
        // 否则
        else{
            // 判断各点之间是否联通
            for(j = 0;j &lt; q - 1;j++){
                if(G[arr[j]][arr[j + 1]] != 1){
                    cout &lt;&lt; "NO" &lt;&lt; endl;
                    break;
                }
                else{
                    // 标记结点被访问
                    flag[arr[j]] = 1;
                }
            }
            if(j &lt; q - 1){
                continue;
            }
            // 判断所有结点是否被访问，可能有些路径会重复访问某个结点，因此要排除这种情况
            for(j = 1;j &lt;= n;j++){
                if(flag[j] == 0){
                    cout &lt;&lt; "NO" &lt;&lt; endl;
                    break;
                }
            }
            // 符合条件
            if(j &gt; n){
                cout &lt;&lt; "YES" &lt;&lt; endl;
            }
        }
    }
}

#include&lt;iostream&gt;
#include&lt;vector&gt;
#include&lt;algorithm&gt;
#include&lt;cstdio&gt;
using namespace std;

bool IsSingle(int *arr , int n , int m){
    int start = 0 , end = m - 1;
    int mid;

    while(start &lt;= end){
        mid = (start + end) / 2;
        if(arr[mid] == n){
            return false;
        }
        else if(n &lt; arr[mid]){
            end = mid - 1;
        }
        else{
            start = mid + 1;
        }
    }
    return true;
}

int main(){

        flag[id1] = id2;
        flag[id2] = id1;

        cin &gt;&gt; arr[i];
    }

        if(flag[arr[i]] == -1){
