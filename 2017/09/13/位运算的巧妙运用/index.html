位运算的巧妙运用

发表于 2017-09-13 | 分类于 算法 | 阅读次数

今天在Leetcode上遇到一道很有意思的题，原题如下:

  1. Single Number II

Given an array of integers, every element appears three times except for one, which appears exactly once. Find that single one.

Note:
Your algorithm should have a linear runtime complexity. Could you implement it without using extra memory?

大概意思是给定一个整数序列，里面的大部分数都重复了三次，只有一个数只出现一次，我们需要找出这个数字。举个例子，1112223334，我们的输出应该是4。要求时间复杂度是O(n)，并且不能使用额外的内存(声明一些简单的变量当然是可以的)。

一开始完全没思路，想了半天也想不出啥东西来。最后无奈的点开了discuss，里面有许多种解决方案，有些简短到只有代码，有些则复杂到让人没有想看的欲望。但其中有一个解答非常棒，答主把数字逻辑的知识运用到本问题上，并且由此给出解答这类题的通用方法。本文就是对这种方法的描述。需要注意的是，读者应该把注意力放在理解算法上，而不要关心算法是怎么想出来的(有时候还是蛮考验智商的= =)。

首先注意到所有重复出现的数的重复次数为3，因此我们设基数(base)为3。对于一个二进制数，表达基数为3的一位整数(即0、1、2)只需要2位。因此我们设两个变量a和b，分别代表高位和低位。另外再设一个进位c。

由于基数是3，因此当ab=10时，加1会使ab变为00。基于这样的想法，假设a和b初始值都为0，那么如果加3 * n个1(n = 0,1,2,…)，那么ab最终为00。

进一步思考，假设所有的整数都是32位的，同时a和b也为32位，并且初始均为0。那么对于一个重复出现3次的整数，我们将其每一位都和a、b的对应位按上述两个公式做运算，做完3次后，a和b仍然均为0。再进一步，对于所有重复出现3次的整数，其和a、b进行运算的最终结果为a和b均为0。而对于只出现一次的整数来说，其和a、b的运算结果包含了该整数的信息。比如，如果整数的某一位为1，如果整数出现一次，那么a和b的对应位为01，出现两次则为10。如果是0，则a和b的对应位始终为0。由此我们可以得到另一个重要的结论，在a、b与所有整数都进行运算后，a | b即为我们要找的整数。比较奇特的是，由于本题中我们要找的数只出现一次，所以最终的b其实就是我们要找的值。

Given an array of integers, every element appears M times except for one, which appears exactly N times. Find that one. (M > N)

我们把大部分整数的重复次数改为重复M次，并且把要找的数的重复次数改为N次，M > N。显然，原题中M=3，N=1。对于任意的正整数M和N，我们都可以按上述思路来设计算法。
