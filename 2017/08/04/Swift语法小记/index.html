Swift语法小记

发表于 2017-08-04 | 分类于 Swift | 阅读次数

本文是对The Swift Programming Language一书的总结和摘录。

一、标准类型

1. 常见类型

Swift内置的常见标准类型为:

  • 整数(Int..)
  • 浮点数(Float、Double…)
  • String
  • Bool

2. 可选类型

可选类型是Swift独有的类型，其有两层含义:

  • 一是变量是否有值(没有值时变量为nil)
  • 二是变量的值是多少

使用问号和感叹号的区别是：

  • 问号声明的可选类型变量，在使用时需要使用感叹号对其进行显式解包
  • 感叹号声明的可选类型变量，在使用时Swift会对其自动进行隐式解包，因此不需要加感叹号

需要注意的是不论是显示解包还是隐式解包，只要对值为nil的变量进行解包就会触发运行时错误！因此，在使用可选类型的变量之前，最好对其进行nil检查。常见的检查方法有:

  • 直接判断变量是否为nil
  • 使用可选绑定(optional binding)

  • 函数可以作为函数参数
  • 函数可以作为函数的返回值

2. 闭包

2.1 闭包的概念

维基百科上是这样阐述闭包的:

在计算机科学中，闭包（英语：Closure），又称词法闭包（Lexical Closure）或函数闭包（function closures），是引用了自由变量的函数。这个被引用的自由变量将和这个函数一同存在，即使已经离开了创造它的环境也不例外。

由此，我们可以知道闭包其实也是一种函数，只不过其引用了自由变量。而所谓的自由变量就是指那些定义在闭包函数外部的变量。一旦一个函数引用了定义在其函数体之外的变量，这个函数就成了一个闭包函数，而所有被引用的自由变量都会和这个函数绑定在一起，如果创造它们的环境消失了但闭包函数没有消失，那么它们就会一直存在。

三、自定义类型

除了使用Swift内置的标准类型和函数类型，我们可以定义属于自己的类型。定义类型的方法和其他语言类似，即：

  • 通过类定义
  • 通过结构体定义
  • 通过枚举定义

我们将在后面详细讨论这些内容。

四、值类型VS引用类型

在Swift中值类型和引用类型是一个重要的知识点，明确哪些类型属于值类型，哪些类型属于引用类型是非常重要的。

属于值类型的有:

  • Int、Float之类的标准类型
  • 通过结构体定义的类型
  • 通过枚举定义的类型

  • 通过类定义的类型
  • 函数类型(包括闭包)

八、类和结构体

在C++中类和结构体除了默认访问权限之外几乎没有区别，但在Swift中类和结构体的区别非常明显。

两者的相同点有:

  • 可以定义属性
  • 可以定义方法
  • 可以定义下标
  • 可以定义构造函数
  • 可以进行扩展
  • 可以实现协议

两者的不同点有:

  • 类可以继承，结构体不能继承
  • 类有析构函数，结构体没有
  • 类型转换允许类在运行时检查和解释一个类实例的类型
  • 类是引用类型，结构体是值类型

接下来我们简单讨论下属性、方法和下标。

1. 属性

从存储值的角度看，Swift属性大致分为存储属性和计算属性。

存储属性就是一般用于存储值的变量或常量，用关键字var或let声明。当我们希望存储属性不在构造过程中被初始化，而是在使用时被初始化，可以使用lazy关键字。存储属性还可以定义属性观察器，用于监测属性的变动。

计算属性不直接存储值，而是通过提供getter和setter，来间接获取或设置其他属性或变量。

需要注意的是，计算属性必须显式声明其类型，否则无法通过编译。另外， setter可省略，计算属性成为只读属性。

从属性归属者的角度看，Swift属性可以分为实例属性和类型属性。

大部分属性都是实例属性，其属于某个实例，不同实例之间不共享实例属性。类型属性属于类型本身，用static关键字声明，其可以被所有实例共享，也可以直接使用而不进行实例化。

2. 方法

方法是定义在类内部的函数，大致分为两种:

  • 实例方法
  • 类型方法

实例方法只能被实例所调用，其内部可以使用self属性。类型方法可以直接调用而不进行实例化，但其内部只能使用类型属性，而不能使用实例属性(这是由于类型方法无法使用self属性)。

3. 下标

下标是访问集合、列表或序列的快捷方式。

九、继承

Swift中只有类可以继承，结构体和枚举都不可以继承。另外，有些编程语言支持多继承，但Swift只支持单继承。其他关于继承的东西这里就不赘述了。

十、构造与析构

1. 结构体

结构体只有构造函数，没有析构函数。

由于结构体无法继承，因此结构体的构造函数相对比较简单，只要所有属性都被初始化即可。另外，结构体均有一个逐一构造函数，这是自动获得的。通过逐一构造器，我们可以直接赋予属性初始化值。

2. 类

2.1 两种构造器

一个类的构造函数(构造器)可以分为两种:

  • 指定构造器
  • 便利构造器

指定构造器是类中最主要的构造器，其将类内所有属性初始化，并根据继承链向上调用父类的构造器完成对父类属性的初始化。每个类都必须至少有一个指定构造器。

便利构造器是类中比较次要的辅助型构造器，其通过调用类中的指定构造器完成构造。在Swift中我们可以使用convenience关键字来声明一个便利构造器。

对于指定构造器和便利构造器，我们需要记住两点:

  • 指定构造器总是向上代理
  • 便利构造器总是横向代理

2.2 两段式构造过程

类的构造分为两个阶段进行:

阶段一

  • 某个指定构造器或便利构造器被调用。
  • 完成新实例的内存分配。此时内存还没有被初始化。
  • 指定构造器确保其所在类中所有存储属性均已被初始化。存储型属性内存完成初始化。
  • 指定构造器调用父类构造器，完成父类属性的初始化。
  • 上述过程沿着继承链一直向上执行，直到顶部。
  • 当到达继承链顶部时，确保所有存储型属性均已被初始化，这个实例的内存被认为已经完全初始化。
  • 阶段一完成

阶段二

  • 从继承链顶部一直往下，每个类的指定构造器均有机会进一步定制实例。此时构造器可以访问self、修改它的属性以及调用实例方法。
  • 每个继承链中的类的便利构造器也可以定制实例和使用self。

2.3 可失败构造器

构造器可以失败，即没有成功初始化一个实例，其返回一个可选类型。

构造器的继承、必要构造器等内容参考The Swift Programming Language

2.4 析构

析构函数只适用于类类型，并且通过deinit关键字声明。当类实例被释放时，Swift将沿着继承链向上调用各个类的析构函数。由于析构函数是自动调用的，因此无需手动调用父类的析构函数。
