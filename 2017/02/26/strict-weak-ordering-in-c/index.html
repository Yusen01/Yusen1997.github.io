Strict Weak Ordering in C++

发表于 2017-02-26 | 分类于 C++

最近刷PAT的时候，有两道题因为段错误死活过不去，调试的时候发现是STL中的sort出现了问题，第一次的时候不知道怎么修改就放弃重写了，第二次又遇到我就很生气了，于是就在网上找了一些资料，最后将问题定位到了strict weak ordering(严格弱排序)上，在这里简单的记录一下。

首先，我们来看一下维基百科上关于严格弱排序的定义：

A strict weak ordering is a binary relation < on a set S that is a strict partial order (a transitive relation that is irreflexive, or equivalently,that is asymmetric) in which the relation “neither a < b nor b < a” is transitive.Therefore, a strict weak ordering has the following properties:
1、For all x in S, it is not the case that x < x (irreflexivity).
2、For all x, y in S, if x < y then it is not the case that y < x (asymmetry).
3、For all x, y, z in S, if x < y and y < z then x < z (transitivity).
4、For all x, y, z in S, if x is incomparable with y (neither x < y nor y < x hold), and y is incomparable with z, then x is incomparable with z (transitivity of incomparability).
The “incomparability relation” is an equivalence relation, and its equivalence classes partition the elements of S, and are totally ordered by <.

严格弱排序就是定义在集合S上的一种二元关系(relation)，这种二元关系就是“<”。这里的小于号不是普通的小于号，它代表的是一种关系，而不是简单的比大小。严格弱排序有四种性质，我们就看前两种吧。第一条，集合中的任何元素x，它跟自身都不满足“<”这种关系。第二条，集合中的两个元素x，y，如果x<y，那么y<x一定不成立。虽然看起来像是废话，但这就是数学的严谨性。

最重要的一点是，如果x<y不成立并且y<x也不成立，写成符号表达式就是!(x<y)&&!(y<x) = true，那么x和y就是不可比的(imcomparable)，根据定义，它们之间就是等价关系。理解了这一点，我们才能理解为什么sort会崩溃。

接下来我们看看STL官方文档对严格弱排序的描述：

A Strict Weak Ordering is a Binary Predicate that compares two objects, returning true if the first precedes the second. This predicate must satisfy the standard mathematical definition of a strict weak ordering.

根据这种描述，严格弱排序就是一个比较两个对象的二元谓词，或者说二元函数，这个函数返回true如果第一个对象先于第二个对象，同时它还要满足严格弱排序的数学定义。因此，如果函数返回true，即如果第一个对象(x)先于第二个对象(y)，我们可以认为x<y关系成立。

有了上面这些描述，我们就来具体看看sort函数。C++中的sort函数可以接受两个随机迭代器和一个比较函数，但规定了比较函数必须是严格弱排序的。sort函数在对序列排序时，必定会判断两个元素的相对位置应该是怎样的，那么sort是如何判断的呢？下面是我的一些理解，由于我还是初学C++，对sort的实现机制不是很了解，因此更多的是我的一些推测和猜想，但也能解释为什么sort函数会崩溃。

假设有两个对象x，y和一个比较函数f(a,b)。为了确定x和y的相对位置，首先计算f(x,y)，如果返回true，说明x<y，如果返回false，说明!(x<y)，注意这里不是y<x。然后计算f(y,x)，如果返回true，说明y<x，如果返回false，说明!(y<x)。x和y之间存在三种关系，x应该先于y，y应该先于x，x和y等价。对于前两种关系，只要一个返回true另一个返回false，sort就能正确地排出它们的顺序，而对于x和y等价，sort就需要两个函数都返回false。而我之前sort函数出现段错误的原因，就是我将比较函数定义成了在x和y等价时返回true。当用这种比较函数比较等价的x和y时，sort函数发现两个函数返回都是true，而这是一种没有定义的情况，可能就会触发某种机制，从而产生段错误。避免这种错误的方法就是在定义比较函数时，一定要确保两个对象等价时返回false。
